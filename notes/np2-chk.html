numpy2 compliance check

numpy 2.0 was released this week. Given the numerous updates, I wanted to test if the code I’d written over the years was still usable. numpy 2.0 migration guide suggests testing with ruff. With all my python code readily available in the m-one repo., this made it pretty easy. All I had to do was to run the following at command line. This command searches for files with numpy calls, and those found are checked for compliance:

grep "numpy" *.py | ruff check --select NPY201

Out of the 22 files, one did not comply. A cursory check revealed that it was not so much a specific change from numpy, but a legacy print statement. It was an easy fix.
