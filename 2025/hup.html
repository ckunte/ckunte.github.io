Blocking internet's underbelly

30 Dec 2025, 2 min

tl;dr: This script updates /etc/hosts file — populating with known bad sites, which prevents scripts and resources from these sites loading up on your computer.

#!/usr/bin/env bash
# hup.sh -- 2025 ckunte
printf "Blocking internet's underbelly from reaching this computer (%s)..." "$(hostname)"
curl -s https://raw.githubusercontent.com/StevenBlack/hosts/master/alternates/fakenews-gambling-porn/hosts | sed 's/^||//' > /etc/hosts
printf " done.\n"

Save the script to a file, hup.sh, and run:

sudo bash ./hup.sh

Some explanation re: hosts file

The Domain Name System (DNS) is the phone book of the internet. Prior to DNS, computer systems contained a hosts file that essentially did the job of the DNS — mapping an IPV4 address to a domain name. This (legacy) file continues to be included in operating systems. It is now also known as DNS hosts file.

  • In UNIX-like systems, it is located at: /etc/hosts
  • In Windows, it is located at: c:\Windows\System32\Drivers\etc\hosts

How to block using a hosts file

This hosts file has the highest priority for mapping IP address on a given computer system. While this is not the best way to track and update the ever-changing IP addresses of domains on the internet today, it has one key benefit. It can be effectively used to block domains, e.g., mapping a domain to 0.0.0.0 instructs the computer to not request any resource from this domain, thereby effectively preventing any from the blocked domain from reaching the computer with this hosts file.

0.0.0.0  somebadwebsite.com
0.0.0.0  anotherbadsite.net
...

The downside of hosts file is that it does not support wildcards, and so all variants including subdomains will need to be explicitly mapped. In comparison, tools like Pi-Hole support wildcard and regular expression-based blocking, which can significantly reduce explicit (and manual) mapping. Crowd-sourced mapping is one way to overcome this handicap of hosts.

How the script helps prepare a hosts file

The script above works as follows:

  1. Downloads a crowd-sourced and aggregated hosts file
  2. Replaces /etc/hosts with the contents of the above

Why run this script periodically

Since crowd-sourcing is a periodic work, it is good to run this script once a month or so to update the /etc/hosts file with the latest blocked domains.
